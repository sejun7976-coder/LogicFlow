## 1. 프로젝트 개요

### 1.1 프로젝트 목표

> 사용자의 독해와 논리적 사고 훈련을 통해 사고력과 문제 정의 능력을 향상시키고, 더 나은 질문을 만들어낼 수 있는 능력을 기르게 도와주는 학습앱

### 1.2 대상 사용자

> 자신의 생각을 논리정연하게 글로 써내려가지 못하는 사람들

### 1.3 핵심 가치

> 1. LLM API를 사용하여 사용자의 문장과 정답과의 일치도 제공
> 2. 안드로이드 네이티브인 STT를 사용하여 답안 입력 방법의 다양성과 편의성 제공

## 2. 기능 요구사항 (Functional Requirements)

> 시스템이 *무엇을 하는가*를 정의한다. 최소 5개 이상 작성
> 문장 패턴: "사용자는 ~할 수 있다" / "시스템은 ~한다"

### FR-01: 다양한 독해 지문 제공

- **설명**: 시스템은 공공 데이터셋(AI Hub 국어 교과 지문형 문제 데이터)을 기반으로 사용자에게 요약 훈련용 지문을 제공
- **수용 조건**:
  - [ ] 사용자가 앱 사용 시 난이도를 선택할 수 있어야 함
  - [ ] 지문을 선택하면 텍스트를 읽기 편한 UI로 확인할 수 있어야 함
  - [ ] 데이터셋에서 정답, 해설을 제외한 필요한 지문만 가져와 호출하는 방식으로 데이터를 로드

### FR-02: STT 기반 요약문 자동 입력

- **설명**: 사용자는 스마트폰의 마이크를 이용해 지문에 대한 자신의 요약을 음성으로 말하고, 시스템은 이를 텍스트로 변환하여 입력
- **수용 조건**:
  - [ ] 화면에 마이크 (녹음 시작/종료) 버튼이 제공
  - [ ] 녹음 중 사용자의 음성이 실시간 또는 녹음 종료 직후 텍스트 필드에 자동으로 변환되어 채워짐

### FR-03: 요약문 수동 편집

- **설명**: 사용자는 STT 인식 오류를 수정하거나 직접 타이핑하기 위해 변환된 텍스트를 편집할 수 있음
- **수용 조건**:
  - [ ] STT로 변환된 텍스트가 표시된 입력창은 사용자가 터치하여 자유롭게 수정할 수 있어야 함

### FR-04: LLM 기반 일치도 분석 및 평가

- **설명**: 시스템은 제출된 사용자의 요약문과 원문에서 추출한 모범 요약문을 LLM API를 통해 비교하고 피드백을 제공
- **수용 조건**:
  - [ ] 단순하게 단어를 매칭시켜 일치도를 분석하는 것이 아니라 문맥 기반(의미론적 일치율)을 계산하여 화면에 표시
  - [ ] 누락된 핵심 키워드 및 문장 개선점에 대한 구체적인 텍스트 피드백이 제공

### FR-05: 대화형 추가 피드백

- **설명**: 사용자는 피드백 결과에 대해 궁금한 점이나 부족한 부분을 LLM과 채팅으로 대화하며 추가로 개선
- **수용 조건**:
  - [ ] 평가 결과 화면에 챗봇 형태의 질의응답 인터페이스가 제공
  - [ ] 지문 내용과 사용자가 쓴 요약문을 볼 수 있는 상태로 LLM이 답변

## 3. 비기능 요구사항 (Non-Functional Requirements)

> *어떻게 동작하는가*를 정의한다. 직접 보이지 않지만 품질을 결정한다.

### 3.1 성능 (Performance)

- LLM API를 이용한 분석 결과는 Gemini Flash를 사용하여 네트워크가 정상이라는 가정 하에 앱의 멈춤 현상 없이 3~5초 정도로 빠르게 작성되어야 함

### 3.2 보안 (Security)

- 외부 API Key (AI Hub API, LLM API)는 앱 코드 내부에 평문으로 하드코딩하지 않고 환경 변수나 안전한 저장소를 통해 관리

### 3.3 사용성 (Usability)

* 긴 글을 읽는 사용자의 눈 피로를 줄이기 위해 텍스트 가독성을 고려한 폰트 및 여백이 적용
* STT 사용 시 권한 요청 프로세스가 사용자에게 명확하게 안내

### 3.4 호환성 (Compatibility)

- (예: Android API 24 이상 지원)
- 안드로이드 네이티브 마이크 센서 및 STT 기능이 정상 동작하도록 Android SDK 버전을 API 24로 설정

### 3.5 기타

- 짧은 기간 안에 최소 기능 제품 구현이 가능하게 핵심기능(지문 제공, STT, LLM을 이용한 평가 및 피드백)을 우선으로 구현
- 추후 국어 지문 뿐 아니라 뉴스 기사, 논문 등 다양한 데이터를 쉽게 연동할 수 있도록 아키텍처를 모듈화로 제작
